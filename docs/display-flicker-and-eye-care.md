# 屏幕频闪 / 护眼 与 节能模式(设计记录)

> 2026-06-17 起。背景:拍使用视频时屏幕老是「显示不全 / 像低刷新」(暗带),由此牵出儿童护眼问题。

## 现象与根因

- **拍视频出现暗带 / 显示不全**:不是固件实现问题,也不是刷新率低。渲染是 `canvas.pushSprite` 整帧双缓冲推出(不撕裂),25fps / 翻页 9fps 对肉眼足够。
- **真因 = 背光 PWM 调光的频闪**:背光不是常亮,是高速「亮一下灭一下」,靠**亮的占比(占空比)**调亮度。相机快门比眼睛快,常拍在「灭」的相位 → 一帧里有亮行有黑行 = 暗带。
- 当前固件:`BRIGHT_ON = 90`(满 255),即占空比 ~35%,**有 65% 时间背光是灭的 → 频闪最深**。

## 频闪 ≠ 靠亮度解决(关键认知)

| | 占空比 | 频闪 | 功耗 |
|---|---|---|---|
| 亮度 90(当前) | ~35% | **最深**(灭得多) | 低 |
| 亮度 255 | 100% = 常亮 | 几乎无 | 高 |

- 拉满亮度(255)能消掉频闪,但那是「常亮」的副作用,**代价是费电 + 暗环境太刺眼**。
- **频闪的本质是 PWM 频率,不是亮度。** 频率够高(kHz 级以上),肉眼任意亮度都无感(相机仍可能拍到,因为相机更敏感)。
- ⇒ **正解:抬高背光 PWM 频率 → 任意亮度无频闪,且几乎不费电**(功耗由占空比/亮度决定,不由频率)。

## 儿童护眼 = 频闪只是其一

影响娃眼睛的**主因排序**:看多久 > 离多近 > 环境光够不够 > 频闪。频闪要治,但「控制时长 + 房间开灯 + 别凑太近」才是大头(也呼应「防沉迷/时长上限」那条待做功能)。

## 设计:护眼模式 / 节能模式(待实现)

**先全局把 PWM 频率抬高,两个模式都无频闪;模式只分亮度 + 熄屏时长:**

| 模式 | 亮度 | 熄屏 | 用途 |
|---|---|---|---|
| 🛡 护眼模式 | 适中偏高(看清、对比舒服) | 温和 | 娃主动玩、房间有灯时 |
| 🔋 节能模式 | 低 | 快熄屏 | 省电 / 电量低 / 待机为主 |
| 🎬(临时)拍摄 | 255 常亮 | 禁熄屏 | 录视频专用,拍完恢复 |

两个模式都**无频闪**(靠高 PWM 频率,不靠拉亮度)。模式存 NVS,设置菜单里切。

## 实现要点(真做时验证,勿拍脑袋)

1. **能否设背光 PWM 频率**:查 M5GFX/M5Unified 的背光 light 配置是否暴露 `freq`,或直接对背光 GPIO 用 `ledcSetup(ch, freqHz, bits)` 抬频(需先确认 Cardputer-ADV 背光引脚 + M5GFX 是否用 LEDC,别和库打架)。
   - 若可设 → 任意亮度无频闪,护眼模式不必拉满亮度。
   - 若不可设 → 退方案:护眼模式 ≈ 高占空比(接近常亮)换「不闪」,接受多点功耗。
2. 现有 `BRIGHT_ON=90 / BRIGHT_DIM=25` 改为按模式取值;新增亮度档常量。
3. 拍摄模式:临时 `setBrightness(255)` + `screenOffMs=0`,退出恢复原值。

## 现状(本次只记录,未改码)

- 固件仍是 `BRIGHT_ON=90`。拍视频应急:临时把 `BRIGHT_ON` 改 255 重编译,或相机侧降快门到 1/30~1/50 + 录 30fps + 给机器补环境光。
- 护眼/节能/拍摄三模式 = 待实现功能。
