# Roadmap / 未来规划

> 长期方向。标注 **[零代码]**(换内容包即可)/ **[需固件]**(要改固件)/ **[内容]**(只需做素材)。

## 一、内容包(持续扩展)

平台的核心是「换包就换主题」——**任何人都能自己做包**(一个包 = 按约定摆好的音频 + 图片帧文件夹,见 `docs/assets-spec.md` / `sd-package.md`)。

- **[内容] 交通工具主题包**(进行中)—— 娃痴迷,优先做。
- **[内容] 跟兴趣走的主题包** —— 后续按娃当下的兴趣做,如 **《我的世界》(Minecraft)** 主题已在清单。
- **[零代码] 语言 / 拼读体系** —— 英音/美音、中文拼音、其它语言,只要套得进 A–Z 引擎。
- **[内容] 教学发音包** —— edge-tts / Azure 童声(`tools/make_tts.py` 已就绪,待 Azure key)。

## 二、新交互模式(需固件,不是换包就行)

- **[需固件] 互动做题模式** —— 把「按键→出内容」升级为「出题→娃按键→判对错 + 反馈」。三种媒介变体:
  - **音频 + 图片**:既给声音又给图(最直观,适合小娃)
  - **纯音频**:只播题目音(听音找键,练听辨)
  - **纯图片**:只显示图(看图找键,练认读)
  - 题面对象可为**字母**(A–Z)或**单词的实体**(动物 / 物品图):如「按出 apple 开头的字母」。
  - 实现:答案判定 + 对错反馈(声/动画)+ 出题逻辑;可作为一种 pack 类型(`pack.txt` 标 `mode=quiz` + 题目映射)。

## 三、显示与护眼

- **[需固件] 护眼模式(播放时主动息屏)** —— 区别于现有「闲置熄屏」:ABC 播放时**先把该字母/单词的图显示一下(几秒)→ 主动息屏 → 音频继续放完**。娃「看一眼 + 专心听」,大幅减少盯屏时间(看多久 = 护眼第一因素)。可设置档 / 包标志。
- **[需固件] 背光 PWM 抬频** —— 任意亮度下肉眼无频闪(详见 `docs/display-flicker-and-eye-care.md`);与护眼模式叠加 = 双重护眼。功耗几乎不增。
- **[需固件] 防沉迷** —— 每日游玩时长上限,到点温柔熄屏(娃已上瘾,真实需求)。

## 四、其它

- **[内容] 内容包分发** —— 各主题包打 zip 挂 GitHub Release,附 CREDITS(OpenMoji / Suno 等三方)。
